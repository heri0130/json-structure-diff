## 2. ⚠️ 가장 중요한 설계 결정 — "구조 비교"

이 도구는 **값 비교가 아니라 구조(키/타입) 비교**다. (세션 중반에 값→구조로 전환했음)

- `added`/`removed` = 키가 추가/삭제됨
- `changed` = 같은 키인데 **값의 타입**이 바뀜 (예: `number → string`)
- `unchanged` = 키가 있고 타입이 같음 — **값이 달라도 동일로 본다** (예: `89000 → 79000`은 동일)
- **배열**은 요소를 **대표 스키마로 병합**해 비교 → 개수·순서·값이 달라도 요소 스키마가 같으면 `unchanged`. 요소에 새 키가 생기거나 타입이 바뀌면 `changed`. (경로 표기: `list[].필드`)

→ 새 기능을 넣을 때 이 "구조 우선" 철학을 깨지 않도록 주의.

## 3. 구현된 기능 (전부 동작)

- **모드 전환** (상단 풀폭 세그먼트): `비교` / `명세서`. 전환 시 입력·결과 초기화.
- **비교 모드**: 변경 전/후 2칸 입력 → `분석하기` → 결과 탭(`변경 사항` / `명세서`)
  - 변경 트리: 색상 + `before(타입) → after(타입)` + 타입 변경 강조, **펼치기/접기**, **변경된 것만 보기** 토글
- **명세서 모드**: JSON 1칸 입력 → `명세서 만들기` → 필드 명세표(경로·타입·예시, 들여쓰기 트리, **펼치기/접기**)
- **JSON 유효성 검사**: 빈/깨진 입력 안내, **JSONC 주석 허용**(`//`, `/* */`; 문자열 안 `//`는 보존)
- **결과 내보내기**: 변경 사항/명세서를 마크다운 표로 클립보드 복사
- **예시 칩**: 비교용 5종(변경 전/후 쌍), 명세서용 5종(단일 JSON, 결제 예시엔 주석 포함)
- **테마 4종**: 라이트/다크/공주님(핑크+하트비)/왕자님(파랑+별비) + 공주님·왕자님 모드 마법봉 커서. localStorage 저장.

## 4. 코드 구조

```
src/
  lib/                 # 순수 로직 (UI 무관, 테스트 대상 — 여기에 로직, UI엔 두지 말 것)
    diff.ts              구조 비교 (diffJson) — 핵심
    spec.ts              명세서 생성 (generateSpec)
    parse.ts             JSON 파싱 (parseJsonInput, JSONC 주석 제거)
    valueType.ts         값의 JSON 타입 판별 (valueType)
    export.ts            마크다운 변환 (diffToMarkdown, specToMarkdown)
  components/
    DiffTree.vue         변경 트리 (재귀 + 펼치기/접기)
    SpecTable.vue        명세서 표 (펼치기/접기)
  composables/
    useTheme.ts          테마 상태·적용·하트/별 입자 (모듈 싱글톤)
  App.vue                화면 조립 (입력·모드·결과·예시·복사)
  style.css              전역 + 테마 토큰([data-theme=...])
```

각 `lib/*.ts`마다 `*.test.ts` 동봉. **새 로직은 lib에 순수 함수로 + 테스트 먼저(TDD)**.
